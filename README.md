# 경제병리학 연구 시스템 (Economic Pathology Research System)

## 개요

경제 위기를 의학적 질병으로 비유하여 체계적으로 분석하고 예측하는 학술 연구 시스템입니다. 1929년부터 2025년까지 12개의 주요 경제 위기를 5가지 병리 유형으로 분류하고, 실시간 데이터를 통한 현재 경제 상태 진단 기능을 제공합니다.

## 시스템 특징

- **의학적 접근법**: 경제 위기를 질병으로 모델링하여 진단-치료-예후 관점에서 분석
- **통합 대시보드**: 9개 차트가 하나로 통합된 마스터 대시보드
- **실시간 진단**: FRED, Yahoo Finance, IMF 데이터를 활용한 현재 경제 상태 실시간 모니터링
- **데이터 신뢰성**: 6단계 검증 시스템으로 연구 데이터의 학술적 신뢰성 보장
- **Raw 데이터 출력**: 연구 목적의 원시 데이터 Excel 출력 (4개 시트)

## 설치 및 설정

### 1. 필수 패키지 설치

```bash
pip install pandas numpy matplotlib seaborn plotly yfinance fredapi requests beautifulsoup4 lxml openpyxl
```

### 2. API 설정

`.env` 파일을 프로젝트 루트에 생성:

```env
# FRED API Key (연방준비제도 경제 데이터)
FRED_API_KEY=""YOUR_API_KEY""

# IMF SDMX API
IMF_SDMX_ENABLED=true

# 설정
DATA_UPDATE_INTERVAL_MINUTES=15
LOG_LEVEL=INFO
```

## 데이터 파이프라인 검증

시스템은 6단계 검증 과정을 통해 데이터 신뢰성을 보장합니다:

1. **데이터 수집 검증**: API 연결 상태, 데이터 완정성 확인
2. **데이터 처리 검증**: 전처리 과정의 정확성 검증
3. **분류 체계 검증**: 병리 분류의 일관성 확인
4. **매핑 검증**: 경제지표-병리상태 매핑 정확성
5. **상관관계 검증**: 통계적 유의성 확인
6. **출력 검증**: 최종 결과물의 무결성 검사

각 단계별로 신뢰도 점수(0-100%)가 계산되며, 전체 파이프라인 신뢰도가 산출됩니다.

## 병리학적 분류 체계

### 5대 병리 유형

1. **순환계 질환** (Circulatory Disorders) - 파란색
   - 유동성 위기, 신용 경색
   - 예시: 1987 블랙먼데이, 1997 아시아 금융위기

2. **대사 질환** (Metabolic Disorders) - 빨간색  
   - 인플레이션, 디플레이션
   - 예시: 1970s 스태그플레이션, 2022 인플레이션

3. **구조 질환** (Structural Disorders) - 주황색
   - 자산 버블, 부채 위기
   - 예시: 1929 대공황, 2000 닷컴버블

4. **면역 질환** (Immune Disorders) - 보라색
   - 시스템 리스크, 전염성 위기
   - 예시: 2008 금융위기, 2020 코로나쇼크

5. **신경 질환** (Neural Disorders) - 초록색
   - 정책 혼란, 기대 불일치
   - 예시: 1998 러시아 위기

## 기술적 특징

### API 연동
- **FRED API**: 연방준비제도 경제 데이터 (실시간)
- **Yahoo Finance**: 주식시장 데이터 (VIX, S&P 500)
- **IMF SDMX**: 국제통화기금 통계 데이터

### 분석 방법론
- **시계열 분석**: 경제지표의 시간별 변화 패턴 분석
- **상관관계 분석**: 병리 유형간 상호연관성 분석
- **클러스터링**: 유사한 위기 패턴 그룹화
- **회귀분석**: 심각도-지속기간-회복률 관계 모델링

## 학술적 기여

1. **새로운 연구 관점**: 경제 위기를 의학적 질병으로 체계화한 최초 시도
2. **정량화 모델**: 경제 위기의 심각도, 지속기간, 회복률 수치화
3. **예측 프레임워크**: 병리학적 진단 기준을 통한 위기 예측 모델
4. **실시간 적용**: API 기반 현재 경제 상황 실시간 진단 시스템

## 면책사항

본 시스템은 학술 연구 목적으로 개발되었으며, 실제 투자나 정책 결정에 사용할 수 없습니다. 모든 진단과 예측은 연구 가설에 기반한 것으로, 실제 경제 상황과 다를 수 있습니다.

## 문제 해결

### 자주 발생하는 문제

1. **API 키 오류**: `.env` 파일의 FRED_API_KEY 확인
2. **패키지 누락**: `pip install -r requirements.txt` 실행
3. **데이터 로딩 실패**: 인터넷 연결 및 API 상태 확인
4. **그래프 출력 오류**: output 폴더 존재 여부 확인

문제가 지속될 경우 다음을 확인:

3. API 키 유효성
